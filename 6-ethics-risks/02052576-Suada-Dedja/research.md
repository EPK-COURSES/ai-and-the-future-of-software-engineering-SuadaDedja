# Research: Risks, Ethics, and Limitations of AI in Software Engineering



## 1. Introduction to AI Limitations in Software Engineering

Lately, AI has completely transformed how we write software. Tools like GitHub Copilot or ChatGPT make it incredibly easy to get quick code suggestions and automate boring, repetitive tasks. However, as coding becomes faster, we are facing serious risks that we didn't have to worry about before. Relying blindly on these models introduces technical, legal, and human challenges that we, as future software engineers, need to take very seriously.



## 2. Technical Limitations and Risks

One of the biggest issues with AI right now is "hallucinations." This happens when an AI tool generates a piece of code that looks perfectly fine and clean, but in reality, it is completely wrong, non-functional, or refers to libraries that do not even exist. If a developer does not test this thoroughly, it can break the whole system.



Another massive concern is security. Because these AI models are trained on massive amounts of public repositories, they often clean and reuse code that already has security flaws. This means AI can easily insert dangerous vulnerabilities into a new project without giving any warning to the developer.



## 3. Ethical and Legal Challenges

From an ethical standpoint, copyright infringement is a huge gray area. Many AI tools were trained on open-source projects without fully respecting or aligning with their original licenses. This creates a big legal risk regarding who actually owns the code and whether a company might get sued for using AI-generated suggestions.



Additionally, data privacy is a major issue. When developers copy and paste proprietary company code into public AI models to debug it, they might accidentally leak sensitive data, making it available for the model to train on and potentially show to other users.



## 4. The Human Factor: Overdependence and Skill Degradation

The most important question we have to answer is: What happens if engineers stop understanding the code they ship? When developers start relying too much on AI, their critical thinking and debugging skills start to decline. If you just press "Tab" to accept every suggestion blindly, finding and fixing complex architectural bugs later becomes almost impossible because you don't actually know how the code works.
This is especially dangerous for junior developers. If beginners use AI to solve every single foundational logic problem or homework assignment, they will struggle to grow into actual experts. They miss out on the learning process that comes from making mistakes and struggling with code, which is exactly how real engineering skills are built.


## 5. Human Supervision and Responsibility
Even though AI tools can increase productivity and save time, they should never operate without human supervision. Software engineers remain responsible for the quality, safety, and ethical impact of the systems they develop. AI can support decision-making, but final responsibility still belongs to humans who understand the code and its consequences.

## 6. The Evolving Role of Software Engineers
In the future, software engineers will transition from traditional code writers to system architects and AI supervisors. The focus will shift from syntax and boilerplate coding to high-level design, system integration, and rigorous verification. Engineers must develop strong critical thinking skills to audit AI-generated solutions and ensure they align with security and business goals.

## 7. Conclusion 
AI is becoming an important part of software engineering, but it is not perfect. Hallucinations, security vulnerabilities, privacy concerns, and skill degradation show that AI still has significant limitations. Instead of replacing engineers, AI should be treated as a powerful assistant that requires careful oversight. The future of software engineering will depend not only on smarter tools but also on responsible developers who understand, verify, and improve the code they create.
